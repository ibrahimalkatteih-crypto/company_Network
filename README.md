يهدف هذا المشروع إلى تصميم وتنفيذ شبكة لمؤسسة صغيرة باستخدام Cisco Packet Tracer، مع تطبيق مجموعة من تقنيات CCNA لإنشاء شبكة منظمة، آمنة، وقابلة للتوسع.
تم تقسيم الشبكة إلى عدة أقسام باستخدام VLANs، مع توفير الاتصال بينها باستخدام Router-on-a-Stick، بالإضافة إلى تطبيق خدمات الشبكة الأساسية وآليات الحماية والإدارة.
مكونات الشبكة
راوتر واحد.
سويتش رئيسي (Core Switch).
أربعة سويتشات فرعية (Access Switches).
خادم (Server).
عدة أجهزة حاسب تمثل أقسام الشركة.
الأقسام هي:
الموارد البشرية (HR)
تقنية المعلومات (IT)
المبيعات (Sales)
المالية (Finance)
شبكة خاصة بالخادم (Servers)
التقنيات المستخدمة
تقسيم الشبكة باستخدام VLANs
تم إنشاء VLAN مستقلة لكل قسم بهدف:
تقليل الـ Broadcast.
تحسين أداء الشبكة.
زيادة مستوى الأمان.
Router-on-a-Stick
تم إعداد منافذ فرعية (Subinterfaces) على الراوتر لتمكين الاتصال بين جميع الـ VLANs باستخدام بروتوكول 802.1Q.
DHCP
تم إنشاء DHCP Pool مستقل لكل VLAN لتوزيع عناوين IP تلقائياً مع تحديد:
عنوان البوابة الافتراضية (Default Gateway).
عنوان خادم DNS.
استثناء عناوين الأجهزة المهمة من التوزيع.
DNS و Web Server
تم إعداد الخادم ليقدم خدمتين:
خدمة DNS.
خدمة Web Server.
بحيث يمكن الوصول إلى الموقع باستخدام اسم النطاق company.local بدلاً من عنوان الـ IP.
SSH
تم تفعيل بروتوكول SSH للإدارة الآمنة لأجهزة الشبكة عن بعد، مع إنشاء مستخدم محلي وتوليد مفاتيح RSA واستخدام الإصدار الثاني من SSH.
EtherChannel
تم تجميع رابطين بين الـ Core Switch وأحد السويتشات الفرعية باستخدام EtherChannel بهدف:
زيادة عرض الحزمة (Bandwidth).
توفير مسار احتياطي في حال تعطل أحد الروابط.
STP
تم تطبيق بروتوكول Spanning Tree Protocol (STP) لمنع حدوث حلقات (Loops) في الطبقة الثانية مع الإبقاء على الروابط الاحتياطية.
ACL
تم إنشاء قوائم تحكم بالوصول (ACLs) لتحقيق سياسة أمنية داخل الشبكة، بحيث:
يمنع قسم الموارد البشرية (HR) من الوصول إلى قسم المالية (Finance).
يسمح لقسم تقنية المعلومات (IT) بالوصول إلى جميع الأقسام لإدارة الشبكة.
